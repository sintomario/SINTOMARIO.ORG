1. Arquitectura del Sistema

1.1 Principios Fundamentales

  • Dominio único: sintomario.org acumula toda la autoridad SEO
  • Sitio 100% estático: HTML plano generado por Python, servido por GitHub Pages
  • Coste cero: ~$9/año solo por el dominio (Cloudflare)
  • Autonomía operacional: Una vez deployado, funciona sin mantenimiento
  • Escalabilidad horizontal: Arquitectura probada hasta 2,500+ nodos

1.2 Estructura de Carpetas

2. Estructura del Corpus

2.1 Entidades Corporales (20)

Cada entidad representa una zona o sistema del cuerpo con atributos:

  • nombre: Nombre común del síntoma/entidad
  • descripcion: Definición médica/lay
  • sistema: Sistema orgánico (digestivo, nervioso, etc.)
  • tcm_elemento: Elemento de Medicina Tradicional China
  • categoria: Clasificación para SEO YMYL
  • aliases: Términos alternativos para búsqueda
  • id: Identificador único para índices

2.2 Contextos Emocionales (20)

Cada contexto representa un estado emocional con atributos:

  • nombre: Nombre de la emoción
  • descripcion: Caracterización del estado emocional
  • herida_emocional: Herida subyacente asociada
  • intensidad: Nivel de intensidad (1-5)
  • categoria: Clasificación emocional
  • id: Identificador único para índices

2.3 Matriz de Combinaciones

El corpus completo genera 400 nodos únicos (20 entidades × 20 contextos), cada uno con:

  • Contenido original y específico
  • 4 perspectivas terapéuticas
  • Práctica de integración de 4 pasos
  • 3 FAQs personalizadas
  • SEO completo (title, description, schema)
  • Índice único permanente (SINTO-XXXX)

3. Sistema de Generación

3.1 Motor Python v4.0

El motor sintomario_motor.py es el corazón del sistema:

Características:

  • Velocidad: Genera 400 nodos en ~1.13 segundos
  • Calidad: 100% de nodos indexables
  • Robustez: 0 errores en build completo
  • Validación: Word count, YMYL detection
  • Reportes: JSON con estadísticas detalladas

3.2 Pipeline de Generación

  1. Load: Carga corpus JSON (entidades, contextos, perspectivas, productos)
  2. Generate: Para cada combinación entidad-contexto:
    • Genera capa de reconocimiento (¿qué es?)
    • Genera capa de contextualización (TCM, sistema)
    • Inyecta 4 perspectivas de autores
    • Genera práctica de integración
    • Genera FAQs dinámicas
    • Calcula metadata SEO
    • Genera schema JSON-LD
    • Crea índice SINTO-XXXX
  3. Render: Aplica template HTML con datos
  4. Write: Guarda index.html + data.json por nodo
  5. Static: Genera sitemap.xml, robots.txt, CNAME, _headers
  6. Report: Guarda build-report.json

3.3 Comandos del Motor

# Dry run (validación sin archivos)
python motor/sintomario_motor.py --dry-run --verbose

# Build completo
python motor/sintomario_motor.py --output ./public --verbose

# Configuración personalizada
python motor/sintomario_motor.py --config custom.config.json

4. Estrategia SEO

4.1 Optimizaciones Implementadas

Robots.txt Allow all + Sitemap Generado automáticamente
Open Graph Title, Description, URL, Type Para sharing social
Twitter Card Summary card Optimizado para Twitter/X

4.2 Estrategia de Contenido YMYL

SINTOMARIO maneja contenido de salud (Your Money or Your Life):

  • Schema Article (NO MedicalCondition) para evitar penalizaciones YMYL
  • Disclaimer médico prominente en cada página
  • Barra flotante permanente con aviso de contenido informativo
  • Referencias a autores reconocidos (credibilidad por asociación)
  • Word count mínimo de 50 para indexación

4.3 Long-Tail SEO

La arquitectura captura búsquedas específicas:

  • "ansiedad en el estómago significado" → SINTO-0162
  • "dolor de espalda emocional biodescodificación" → SINTO-0114
  • "cefalea tensional louise hay" → SINTO-0100

6. Sistema de Índices SINTO-XXXX

6.1 Especificación del Índice

  • Formato: SINTO-XXXX (4 dígitos, cero-padded)
  • Rango lecturas: SINTO-0100 a SINTO-0499
  • Fórmula: 100 + (zona_index × 20) + contexto_index
  • Unicidad: Cada combinación entidad-contexto tiene índice permanente

6.2 Ejemplos de Índices

SINTO-0100 = cabeza (0) + bloqueo (0)
SINTO-0101 = cabeza (0) + frustracion (1)
SINTO-0120 = garganta (1) + bloqueo (0)
SINTO-0320 = rodilla (11) + bloqueo (0)
SINTO-0499 = vientre (19) + dependencia (19)

6.3 Índice Maestro

Sistema extendido para futura expansión:

  • 0001: Home
  • 0010-0024: Zonas Hub
  • 0030-0049: Contextos Hub
  • 0100-0499: Lecturas principales
  • 1000-1099: Terapias
  • 1100-1109: Autores
  • 9000+: Páginas especiales
