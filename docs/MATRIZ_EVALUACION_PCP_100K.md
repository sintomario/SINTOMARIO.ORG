# 🎯 Matriz de Evaluación PCP para Corpus 100K Nodos
## Sistema de Validación por Fases - Percepción, Contexto, Permiso

**Versión:** 1.0  
**Fecha:** 25 de Marzo, 2026  
**Objetivo:** Asegurar que cada uno de los 100k nodos implemente correctamente las **3 Fases PCP** (Percepción, Contexto, Permiso) con coherencia narrativa y efectividad persuasiva.

---

## 🧠 **FILOSOFÍA DEL MODELO PCP EN SINTOMARIO**

> **"El PCP no es manipulación; es compasión estructurada que guía hacia la autonomía"**

### **Principios Rectores del PCP en Salud Somática**

1. **Percepción (P):** No diagnosticar desde arriba, sino **validar la experiencia** del usuario
2. **Contexto (C):** No imponer un marco, sino **ofrecer comprensión** que empodere
3. **Permiso (P):** No presionar a la compra, sino **liberar de culpa** para que elija consciencia

### **Integración con Autoridad Triple**
- **Percepción** → Autoridad Legal-Racional (observación clínica)
- **Contexto** → Autoridad Carismática (comprensión empática)
- **Permiso** → Autoridad Tradicional (sabiduría atemporal)

## 📊 **MATRIZ DE EVALUACIÓN PCP - 90 PUNTOS TOTALES**

### **FASE 1: PERCEPCIÓN (30 puntos)**
*Captar atención con diseño terapéutico compasivo y validación inmediata*

#### **1.1 Validación Inmediata del Síntoma (10 puntos)**
```
✅ 10pts: El nodo comienza con validación profunda que nombra el síntoma específico del usuario
         Ejemplo: "Ese dolor en el pecho que aparece cuando todo es demasiado..."

✅ 8pts:  Reconoce el síntoma con empatía clara
         Ejemplo: "Entiendo que sientes presión en el pecho..."

✅ 6pts:  Menciona el síntoma sin validación emocional profunda
         Ejemplo: "El dolor en el pecho puede tener múltiples causas..."

✅ 3pts:  Referencia vaga al síntoma
         Ejemplo: "Los problemas de pecho son comunes..."

✅ 0pts:  Sin validación del síntoma específico (entrada genérica)
```

#### **1.2 Captura de Atención Terapéutica (10 puntos)**
```
✅ 10pts: El hook inicial crea conexión visceral inmediata combinando síntoma + emoción
         Ejemplo: "Tu cuerpo grita lo que tus palabras no pueden decir..."

✅ 8pts:  El inicio genera curiosidad genuina sobre la conexión somática
         Ejemplo: "¿Sabías que tu cuerpo almacena emociones en tejidos específicos?"

✅ 6pts:  El inicio es informativo pero no genera conexión emocional
         Ejemplo: "El pecho es una zona con múltiples conexiones nerviosas..."

✅ 3pts:  El inicio es puramente técnico sin gancho emocional
         Ejemplo: "La anatomía del tórax incluye..."

✅ 0pts:  Sin hook claro o inicio genérico
```

#### **1.3 Establecimiento de Confianza Visual (10 puntos)**
```
✅ 10pts: Diseño inmediatamente terapéutico, colores calmantes, tipografía legible
         - Paleta: verdes/percepción (10b981), espaciado respiratorio
         - Sin elementos agresivos o comerciales intrusivos

✅ 8pts:  Diseño profesional con elementos terapéuticos claros
         - Layout limpio, colores coherentes con fase

✅ 6pts:  Diseño funcional pero sin elementos terapéuticos específicos
         - Estándar pero no memorable

✅ 3pts:  Diseño genérico o potencialmente estresante
         - Colores agresivos, layout denso, comerciales visibles

✅ 0pts:  Diseño que genera desconfianza inmediata
         - Spam-like, colores alarmantes, estructura confusa
```

### **FASE 2: CONTEXTO (30 puntos)**
*Educar sobre el patrón somático con narrativa compasiva*

#### **2.1 Explicación del Patrón Sistémico (10 puntos)**
```
✅ 10pts: Explicación completa del eje síntoma-sistema-emoción con evidencia clara
         - Muestra la cadena causal: "Ansiedad → SNS activado → Contractura muscular → Dolor"
         - Incluye al menos 2 referencias científicas o tradicionales

✅ 8pts:  Explicación clara del patrón con buena estructura lógica
         - Cadena causal presente pero sin profundidad máxima
         - Al menos 1 referencia sólida

✅ 6pts:  Explicación básica del patrón
         - Menciona conexión pero sin desarrollo causal
         - Sin referencias o referencias débiles

✅ 3pts:  Explicación superficial o fragmentada
         - Conexión mencionada pero no explorada

✅ 0pts:  Sin explicación del patrón sistémico
         - Aislamiento del síntoma sin contexto
```

#### **2.2 Validación de la Experiencia del Usuario (10 puntos)**
```
✅ 10pts: Validación profunda que normaliza la experiencia y elimina patologización
         Ejemplo: "No estás roto; estás respondiendo exactamente como está diseñado tu sistema nervioso..."

✅ 8pts:  Validación clara que reduce culpa y vergüenza
         Ejemplo: "Lo que sientes es una respuesta natural de supervivencia..."

✅ 6pts:  Validación básica presente
         Ejemplo: "Es normal sentir esto..."

✅ 3pts:  Validación superficial o implícita
         Ejemplo: "Muchas personas experimentan esto..."

✅ 0pts:  Sin validación de la experiencia
         - Enfoque puramente clínico sin dimensión humana
```

#### **2.3 Educación Terapéutica sin Dogmatismo (10 puntos)**
```
✅ 10pts: Ofrece múltiples perspectivas (médica, tradicional, psicológica) sin imponer una única verdad
         - Presenta opciones: "Desde la medicina china... Desde la neurobiología... Desde la experiencia..."
         - Lenguaje invitatorio: "Podrías explorar..."

✅ 8pts:  Educación sólida con tono respetuoso y abierto
         - Explica sin imponer
         - Ofrece perspectiva principal + alternativas

✅ 6pts:  Educación informativa pero con sesgo hacia una perspectiva
         - Explica bien pero sin pluralidad de enfoques

✅ 3pts:  Educación limitada o sesgada
         - Perspectiva única sin reconocer otras visiones

✅ 0pts:  Dogmatismo o imposición de única verdad
         - "La única causa es..." / "Debes hacer..."
```

### **FASE 3: PERMISO (30 puntos)**
*Facilitar acción sin presión, liberando de culpa*

#### **3.1 Liberación de Culpa por el Síntoma (10 puntos)**
```
✅ 10pts: Liberación explícita y profunda de culpa, vergüenza o autocrítica
         Ejemplo: "No tienes que 'arreglarte'. No hay nada roto en ti. Mereces cuidado, no corrección..."

✅ 8pts:  Reducción clara de culpa con mensaje empoderador
         Ejemplo: "Tu síntoma no es falla; es señal. No requieres arreglo; requieres escucha..."

✅ 6pts:  Mensaje que reduce culpa de manera implícita
         Ejemplo: "Todos necesitamos apoyo a veces..."

✅ 3pts:  Intento débil de reducir culpa
         Ejemplo: "No te sientas mal por esto..."

✅ 0pts:  Sin componente de liberación de culpa
         - Mensaje que podría aumentar culpa: "Deberías haber..."
```

#### **3.2 Posicionamiento del Autocuidado como Inversión Válida (10 puntos)**
```
✅ 10pts: Reframing elegante que posiciona bienestar como inversión esencial, no gasto
         Ejemplo: "Invertir en comprender tu cuerpo es la inversión más inteligente que puedes hacer..."
         - Sin presión, sin urgencia artificial

✅ 8pts:  Mensaje claro que valida inversión en bienestar
         Ejemplo: "Mereces herramientas de calidad para tu sanación..."

✅ 6pts:  Sugerencia básica de valor del autocuidado
         Ejemplo: "Cuidarte es importante..."

✅ 3pts:  Mensaje comercial sin reframing sofisticado
         Ejemplo: "Compra este producto para sanar..."

✅ 0pts:  Sin posicionamiento de autocuidado
         - O peor: mensaje que trivializa la inversión personal
```

#### **3.3 Oferta de Siguiente Paso sin Presión (10 puntos)**
```
✅ 10pts: CTA (Call to Action) invitatorio, opcional, sin urgencia artificial
         Ejemplo: "Si resonates con esto, podrías explorar... Cuando estés listo..."
         - Múltiples opciones de profundidad (lectura, práctica, comunidad, producto)
         - Sin countdowns, sin escasez falsa, sin manipulación

✅ 8pts:  CTA claro pero respetuoso de ritmo del usuario
         Ejemplo: "Cuando te sientas preparado, considera..."

✅ 6pts:  CTA estándar sin presión excesiva
         Ejemplo: "Haz clic aquí para saber más..."

✅ 3pts:  CTA comercial típico
         Ejemplo: "¡Compra ahora! Oferta limitada..."

✅ 0pts:  Sin CTA o CTA agresivo/manipulador
         - "¡Última oportunidad!" / "¡Ahora o nunca!"
```

## 🔗 **MATRIZ DE COHERENCIA ENTRE FASES (15 puntos adicionales)**

### **Transiciones Fluidas (5 puntos)**
```
✅ 5pts: Transiciones entre fases son invisibles, narrativa fluye naturalmente
         - El usuario no nota cambio de "modo", solo profundización

✅ 4pts:  Transiciones claras y coherentes
         - Cambios de fase son suaves

✅ 3pts:  Transiciones presentes pero detectables
         - Se nota cambio de enfoque

✅ 1pts:  Transiciones abruptas o forzadas
         - Cambios bruscos de tono/tema

✅ 0pts:  Sin coherencia entre fases
         - Parecen 3 artículos diferentes pegados
```

### **Progresión Lógica (5 puntos)**
```
✅ 5pts: Progresión natural: validación → comprensión → empoderamiento
         - Cada fase construye sobre la anterior

✅ 4pts:  Progresión clara con buen desarrollo
         - Secuencia lógica evidente

✅ 3pts:  Progresión presente pero inconsistente
         - Algunos saltos en la lógica

✅ 1pts:  Progresión confusa o invertida
         - Orden de fases no óptimo

✅ 0pts:  Sin progresión discernible
         - Fases aisladas sin conexión
```

### **Consistencia de Tono (5 puntos)**
```
✅ 5pts: Tono coherente a través de las 3 fases (compasivo, respetuoso, empoderador)
         - El "personaje" del nodo es consistente (Guía Sabio)

✅ 4pts:  Tono mayormente consistente
         - Pequeñas variaciones pero cohesión general

✅ 3pts:  Tono variable entre fases
         - Cambios notables de voz/narrador

✅ 1pts:  Tono inconsistente
         - Contradicciones de enfoque

✅ 0pts:  Tono fragmentado o contradictorio
         - Parece escrito por diferentes autores
```

    def evaluate_context_phase(self):
        """Evalúa Fase 2: Contexto"""
        # Análisis de explicación de patrón
        pattern_score = self.analyze_pattern_explanation()

        experience_score = self.analyze_experience_validation()

        self.scores['permission']['self_care_positioning'] = care_score

    def evaluate_coherence(self):
        """Evalúa coherencia entre fases"""

        # Análisis de progresión lógica

        self.scores['coherence']['transitions'] = transitions_score
        self.scores['coherence']['progression'] = progression_score
        self.scores['coherence']['tone_consistency'] = tone_score
        
        return sum(self.scores['coherence'].values())
    
    def calculate_total_score(self):
        """Calcula puntuación total PCP"""
        perception = sum(self.scores['perception'].values())
        context = sum(self.scores['context'].values())
        permission = sum(self.scores['permission'].values())
        coherence = sum(self.scores['coherence'].values())
        
        self.total_score = perception + context + permission + coherence
        return self.total_score
    
    def determine_classification(self):
        """Determina clasificación PCP"""
        score = self.total_score
        
        if score >= 95:
            self.classification = "🟢 EXCELENTE - PCP maestro"

## 🚀 **PROTOCOLO DE IMPLEMENTACIÓN PCP**

### **Paso 1: Evaluación Piloto (Semana 1)**
```bash
# Evaluar muestra representativa de 500 nodos
python pcp_evaluate_batch.py --start=0 --end=500 --detailed

# Analizar distribución de scores
python analyze_pcp_distribution.py --input=pilot_pcp_results.json

# Ajustar templates basado en resultados
python calibrate_pcp_templates.py --feedback=pilot_analysis.json
```

### **Paso 2: Evaluación Masiva (Semanas 2-6)**
```bash
# Procesar en lotes de 2,000 nodos
for batch in {0..49}; do
  start=$((batch * 2000))
  end=$((start + 2000))
  python pcp_evaluate_batch.py --start=$start --end=$end --output=pcp_batch_$batch.json &
done

# Consolidar y analizar
python consolidate_pcp_results.py --batches=50 --output=full_corpus_pcp.json
```

### **Paso 3: Optimización Selectiva (Semanas 7-8)**
```bash
# Identificar nodos débiles
python identify_weak_pcp.py --input=full_corpus_pcp.json --threshold=75

# Generar recomendaciones de mejora
python generate_pcp_improvements.py --weak_nodes=weak_pcp_nodes.json

# Optimizar nodos marginalmente deficientes
python optimize_pcp_nodes.py --input=weak_pcp_nodes.json --output=optimized_nodes.json
```
